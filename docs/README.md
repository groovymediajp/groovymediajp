# docs/ - 開発ドキュメント

このディレクトリには、GroovyMedia プロジェクトの詳細な技術ドキュメントをまとめます。

## 1. ディレクトリ詳細

- `app/` : Next.js のルーティング、グローバル CSS、API エンドポイント
- `components/` : UI コンポーネント（記事、フォーム、レイアウト等）
- `content/` : サイト内コンテンツ（md, js）
- `libs/` : ユーティリティや外部連携用スクリプト
- `public/` : 静的ファイル（画像・SVG 等）

## 2. コーディング規約（例）

- 命名規則: キャメルケース（例: `myComponent`）
- ファイル名: React コンポーネントはパスカルケース（例: `MyComponent.js`）
- インデント: 2 スペース
- Lint: `yarn lint` でチェック
- コメント: 必要に応じて JSDoc 形式推奨

## 3. API 設計方針（例）

- API エンドポイントは `app/api/` 配下に配置
- RESTful 設計を基本とする
- レスポンスは JSON 形式
- エラーハンドリングは HTTP ステータスコードとメッセージで返却

## 4. FAQ

- Q. 開発サーバーが起動しない場合は？

  - A. 依存パッケージを再インストール（`yarn install`）し、再度 `yarn dev` を実行してください。

- Q. 新しいコンポーネントを追加したい場合は？
  - A. `components/` 配下に用途ごとにディレクトリを作成し、パスカルケースでファイルを追加してください。
